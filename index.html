Basic data types and operators

  • Data types:
    • Integers (1, -24, 190)
    • Floats (1.0, 3.14159, -14.2)
    • Complex (3 + 2i)
    • Booleans (True, False)
    • Strings ("abc", "this is a string")
  • Operators:
    • Arithmetic (+, -, *, **, /, //, %)
    • Logic (and, or, not)
    • Comparison operators (<, <=, >, >=, ==, !=)
    • Special (in, is)

Variables

  • Containers for storing data
  • No declaration + initialization in Python
  • No specific command for creating a variable
  • Several rules for variable names (alphanumeric, underscores, cannot start with a number)
  • Variable names are case-sensitive (var != Var)

Data Structures

  • list: Collection of values, ordered, mutable ([1, 2, 3])
  • tuple: Collection of values, ordered, immutable ((1, 2, 3))
  • set: Collection of values, unordered, mutable, no duplicates ({1, 2, 3})
  • dict: Collection of key-value pairs, unordered, mutable, no duplicate keys ({'a': 1, 'b': 2})

Control Flow

  • if-elif-else statement: One of the branches is executed based on a condition
  • for statement: Iterates over a sequence
  • while statement: Iterates as long as a condition is met
  • continue statement: Continues with the next iteration of the loop
  • break statement: Stops a while or for loop.
  • pass statement: Does literally nothing

Functions

  • Blocks of code that are only executed once called
  • Can accept data that the inner code works with (parameters)
  • Parameters can be ordered (f(1, 2, 3)) or named (f(a=1, b=1)), and also combination of both (f(1, 2, c=3))
  • However, named parameters have to be after ordered parameters (f(a=1, 2, 3) throws an error)
  • Can return data that the outer code works with
  • Functions can also be anonymous with a slightly different syntax ((lambda a,b: a+b)(1, 2))

Output formatting

  • C-style formatting with %
    • %s for a string
    • %f for a float
    • %.2f for a float with 2 digits after the floating point
    • %d for an integer
  • format function
    • "I'm {} years old".format(24)
    • {} will be replaced with arguments to format
    • Can also used named arguments as "{name} is {age} years old".format(name="Vojtech", age=24)
  • f-strings
    • Prefix a string with an f (f"string")
    • Then variables can be put directly into the string using brackets (f"My age is {age}")
    • Otherwise formatting similar to format function

Greatest exponent

  • Description: Write a function called find_exp that, provided two parameters n and lim, will find the greatest exponent n can be powered to that will make the result (n**m) lesser or equal tha lim. Then, return such number. In other words, find the largest m for which the equation nm <= lim holds true, and return m. If lim is less than 1, you don't have to bother with negative results, simply return None.
  • Inputs:
    • n: The n in the equation nm <= lim
    • lim: The lim in the equation nm <= lim
  • Expected output: The highest possible exponent m that satisfies the equation nm <= lim.
  • Example: 
    >>> find_exp(2, 500)
    > 8
    >>> find_exp(2, 512)
    > 9
    >>> find_exp(2, 1)
    > 0
    >>> find_exp(2, 0)
    > None

Word frequency

  • Description: Write a function called word_freq that, provided a parameter sentence will create a dictionary of key-value pairs where key is a word that is in the sentence and value is the number of occurrences of the word in the sentence. The program should be case-insensitive, i.e. "horse" and "Horse" are considered the same word.
  • Inputs:
    • sentence: The sentence that will be split into words and then word occurrencies will be counted.

Caesar cipher

  • Description: Write two functions called cipher and decipher. The functions should encrypt and decrypt a text supplied in the parameter text using the Caesar cipher with left shift. The function cipher accepts two arguments: text as advertised earlier and an argument n representing the number of characters to shift the text's characters by. The function decipher expects the same arguments. Note that n is always expected to be positive for both functions.
  • Inputs:
    • text: A sentence (string) that is to be ciphered/deciphered.
    • n: Number of characters to shift the text's characters by.
  • Expected output: For cipher, a string encoded by the Caesar cipher with shift of n. For decipher, a decoded string (given that the same n was used for both encoding and decoding).
  • Example: 
    >>> cipher("abcd", 1)
    > "zabc"
    >>> decipher("zabc", 1)
    > "abcd"
    >>> cipher("hello", 3)
    > "ebiil"
    >>> decipher(cipher("hello", 10), 10)
    > "hello"

Collatz conjecture

  • Description: Write a function called collatz that, provided an argument n, will return a sequence of numbers as specified by the Collatz conjecture for given n. Collatz conjecture can be summed up by the following statement: if given number is even, divide it by two, if it's odd, triple it and add one. Repeating this generates a sequence that should eventually end up at 1, and that's when the sequence stops.
  • Inputs:
    • n: The first number in the sequence generated by Collatz conjecture. Expected to be positive.
  • Expected output: A list containing all the numbers generated by the Collatz conjecture, starting with n.
  • Example: 
    >>> collatz(5)
    > [5, 16, 8, 4, 2, 1]
    >>> collatz(3)
    > [3, 10, 5, 16, 8, 4, 2, 1]
    >>> collatz(1)
    > [1]
    >>> collatz(4)
    > [4, 2, 1]

Libraries

  • A set of functions one can import and use
  • The amount and variety of Python's libraries is probably the largest any programming language offers
  • Python has a standard library that is distributed with Python and offers access to the basic functionality
  • Other libraries have to be specifically downloaded
    • PyPI (pip3 install name_of_the_library)
    • Conda (conda install name_of_the_library)
  • There are multiple ways to access the functionality of a library
    • import name_of_the_library and then name_of_the_library.function_name
    • from name_of_the_library import function_name and then function_name
    • from name_of_the_library import * and then function_name

Creating your own library

  • Python recognizes libraries by the existence of a file called __init__.py
  • As long as such a file exists in given folder, you can import functions from other files in that directory
  • Any directory in your path variable that contains a __init__.py file is treated by Python as a library
  • The __init__.py can even be totally empty
  • The __init__.py can export other modules inside the file, but that is usually the case for more complicated libraries
  • A function called something from a file called example.py can then be imported as from example import something

Popular (relevant) libraries

  • numpy: A library for mathematics with high-dimensional arrays/matrices. One of the biggest advantages is the speed it offers.
  • scipy: Offers functions for scientific computing, linear algebra, optimization, etc. Built on numpy, hence also fast.
  • scikit-learn: Machine Learning library made on top of scipy.
  • pandas: Great for data manipulation and analysis.
  • plotly: Data visualization and analytics. Aside from plots offers also dashboards.
  • pillow: Library for working with images.
  • requests: HTTP requests made very simple. Useful when working with APIs.
  • tensorflow: Machine Learning library that offers a very low-level approach towards building Neural Networks
  • keras: A high-level API for tensorflow. Gains in simplicity what it lacks in flexibility.
  • pytorch: Another Machine Learning library with a slightly lower-level approach than tensorflow.

numpy

  • numpy.array is handled very differently to Python's list under the hood, making numpy.array significantly more efficient
    • The array carries meta information about its shape separately and thus, operations that change the arrays shape only require to change that one meta information without touching the actual data
    • For details, see this link: https://ajcr.net/stride-guide-part-1/
    • The implementation comes with some limitations, e.g. all elements of an array have to be of the same type
  • Numpy offers a lot of mathematical functions: https://numpy.org/doc/stable/genindex.html

Overview of plotting libraries

  • matplotlib
    • The most popular visualization library in Python
    • Partly thanks to the fact that it has been around for more than a decade
    • Very easy to generate simple plots, but gets a bit tedious when more complex visualizations are needed
  • plotly
    • Offers interactive plots
    • Plotly Express for basic visualizations and Plotly Graph Objects for more complex situations
    • Can be scaled into a web-based interactive dashboard using Plotly Dash
  • seaborn
    • High-level interface for matplotlib
    • Some complex visualizations can be made very easily, however often it is necessary to know both seaborn and matplotlib
  • ggplot, altair, bokeh, pygal, missingno

Plotly

  • Works both with pandas DataFrames and numpy arrays
  • Express
    • Provides a high-level API to Plotly
    • Some complicated visualizations can be done very easily
    • Some features are not there at all and one has to opt for Graph Objects
  • Graph Objects
    • More flexible than Express, but usually requires more code to be written
    • Graph Objects have been around for longer hence it is easier to find help online
  • Dash
    • Provides a way to create web-based interactive dashboards

Browsing directories

  • Browsing directories can be done with os.listdir or os.scandir
    • os.listdir returns list of names of the files in given directory
    • os.scandir returns an iterator of os.DirEntry which offer access to some extra attributes than just the name
  • For filtering files by for example their suffixes, one has to use the re library to do pattern matching
  • The glob library uses re under the hood so we don't have to do filtering manually
  • The glob library can also search for files in sub-directories

Handling files

  • A file can be opened simply with the open function
  • The open function accepts an argument mode
    • mode='w' indicates that we want to write to a file, and it will even create an empty file for us
    • mode='r' indicates that we want to read from a file, and will return an error if the file does not exist
  • Once we open a file, we can write to it using .write and read using .read, .readline and .readlines
  • Once we finish reading/writing, we need to close the file using .close
  • The with keyword handles closing the file for us

Pandas

  • Library for working with files (CSV, XML, JSON, ...)
    • List of accepted files and their respective functions can be found here
  • Two basic data types:
    • pandas.Series: represents a vector
    • pandas.DataFrame: represents a matrix, essentially multiple pandas.Series
  • Unlike with numpy.array and Python's list, pandas.DataFrame can have named columns and rows
    • The default names are numbers
    • One can pass the argument columns to change the names of the columns
    • One can pass the argument index to change the names of the rows
    • Both column and row names can also be changed by passing a dictionary instead of a list

Accessing data

  • The .head and .tail functions offer a preview of the DataFrame
  • The .index and .columns attributes show the name of the rows and columns, respectively
  • Columns, rows and their intersections can be accessed by using .loc, .iloc or simply using the bracket notation
    • Accessing one column will result in a pandas.Series
    • Accessing multiple columns will result in a pandas.DataFrame
    • Bracket notation on pandas.Series works with rows, while with pandas.DataFrame, it works with columns
    • .loc and .iloc work both with columns and rows
    • .loc works with row and columns names, .iloc works with their position

Boolean indexing

  • Works similarly as with numpy.array
    • A condition on a DataFrame results in a DataFrame of boolean values
    • Boolean DataFrame works like a boolean masks
  • Aside from normal Python conditions, pandas offers several methods for obtaining boolean masks:
    • .isin: Works like Python's in, but can be used on a whole DataFrame or Series
    • .isna: Works similarly as Python's is None, but can be used on a whole DataFrame or Series
    • .str: Offers access to Pandas' string methods, including for example .startswith or .contains

Missing data

  • .isna returns a boolean indicating whether a cell contains missing data
  • .dropna drops any rows or columns that contain a cell with a missing value
    • axis=0 will drop rows
    • axis=1 will drop columns
    • One can also specify if one missing value is enough to perform a drop, or all values have to be missing, using how="any" or how="all", respectively
  • .fillna replaces missing values with given value
    • We can also also specify that we want to use previous/next value to fill the missing one
  • .interpolate fills missing value using an interpolation method
    • Defaults to linear interpolation, but one can specify a different one, e.g. quadratic

Grouping and merging

  • We can group categorical values using .groupby and apply a function to produce new values using .agg
  • We can also concatenate multiple DataFrames using .concat
  • We can perform the SQL-like join using .merge
    • By specifying how, we control which join will be used (inner, outer, left, right)

sqlite3

  • For local work with databases (no need to setup a server)
  • Once a connection to the database is established, we can just use SQL commands to work with it
  • SELECT returns a list of tuples, which makes it easy to operate with the data in Python
  • For more complex usage, see this link

Object-oriented programming

  • Modern programming focuses on OOP
  • Objects are instances of a class - a blueprint with given properties
  • Classes can inherit properties of other classes (for example a square can inherit properties of a rectangle)
  • As soon as we create an object, a constructor function __init__ is called
    • Whatever arguments do we pass to the object, they are further passed to the constructor
  • Variables can be stored and accessed in an object using the self keyword
  • We can also directly access an object's variables and methods by calling .name_of_the_variable
  • Class inheritance allows us to implement only the functions we want to override and keep the rest
  • Overriding specific methods allows us to perform math operations directly with the object
    • The complete list of those methods can be found here

Application Programming Interface

  • Multiple kinds of APIs:
    • SOAP: Messages exchanged in XML
    • XML-RPC: Uses XML, defines a number of data types
    • JSON-RPC: Similarly to XML-RPC, bud uses JSON
    • REST: Commands defined by HTTPs GET, POST, PUT and DELETE. Can use multiple standards for the message itself.
  • In Python, we can use several libraries to make requests to APIs:
    • requests: A high-level interface for making requests
    • urllib2: A lower-level interface
    • aiohttp: An interface for asynchronous requests
  • It is also possible to define an API of our own very quickly:
    • Flask API
    • Flask
    • Django REST Framework

requests

  • The documentation can be found here
  • The requests library has a designated method for each GET, POST, PUT and DELETE request
  • The params argument handles parameters that will be shown in the URL:
    • requests.get('example.com', params={'par1': 'abc', 'par2': 'bcd'}) will be transformed into the example.com?par1=abc&par2=bcd URL
    • Having parameters shown in the URL has its upsides, but also its downsides
  • The data argument handles parameteres that will NOT be shown in the URL:
    • requests.get('example.com', data={'par1': 'abc', 'par2': 'bcd'}) will result into the example.com URL while still carrying the data
    • For example for sending passwords over the internet, this is obviously a better option
  • The headers argument allows you to send custom HTTP headers:
    • requests.get('example.com', headers={'content-type':'application/json'} will specify that you're expecting the data to be returned in json
    • Headers can be especially useful when an API has some limitations (e.g. max 100 requests per minute)

Third-party APIs

  • Usually, an API comes with its documentation, otherwise the user would not be able to work with it
  • And every API also comes with a number of limitations, usually listed in the docs.
  • Moreover, some APIs require you to login to your account before being granted the access, which can be done via several headers:
    • X-API-KEY
    • Authorization
    • X-Authentication-Token
  • Few examples:
    • Github API limits the number of results you can get with one call to 100 and up to 5000 requests per hour
    • Zenhub API allows up to 100 requests per minute
    • Clockify API allows up to 10 requests per second and also limits the number of results you can get per call

Dealing with limitations of APIs

  • Github and Zenhub APIs return several headers to make working with the limitations easier, however it's not an obligation for all APIs:
    • X-RateLimit-Limit: How many requests can I make per given time
    • X-RateLimit-Remaining: How many requests can I currently make (i.e. how many are left until reaching the limit)
    • X-RateLimit-Used: How many requests have I made
    • X-RateLimit-Reset: The time when the limit will reset
  • Provided these headers, one can simply check with every request whether the limit has been reached, and then wait until it is refreshed
  • If the headers aren't available, one can check for the status_code of the request, which indicates whether the request has finished successfully (status_code == 200), or it was forbidden to request anything (status_code == 403)
    • This method is a bit less reliable as we can either check for status_code == 200, which might not take into account e.g. forgetting to log in (401) and then we enter an infinite loop, or we can check for status_code == 403, which then allows 401 requests to pass through and we obtain damaged data
    • Moreover, we are making unnecessary requests until we get 200

Creating an API

  • Flask API allows for very straightforward API creation
  • You can simply use @app.route('/the/url', methods=['GET', 'POST', ...]) above a function to indicate that given function should be called when /the/url is requested
  • request.method is used to carry which method was used with the request
  • request.data carries the data passed in data
  • request.args carries the data passed in params
  • Whatever then gets returned by the function is passed as the data of the response

API wrappers

  • Many APIs come with a Python library that is essentially a wrapper to make things easier
  • However, some specific cases might not be implemented in the wrapper, and then we still need to go for the requests library
  • Examples:
    • PyGithub for the Github API
    • yfinance and yahoo_finance_api2 for the Yahoo Finance API
    • wbpy for the Worldbank API
    • And many more

Web scraping

  • Essentially making a request to a website to retrieve its source code and then parsing it
    • Meaning that this can be done with the requests library
    • However, this is rather ineffective and there are more appropriate tools for scraping with Python
  • Having some knowledge of HTML definitely makes the job easier
  • Example of some scraping libraries:
    • BeatifulSoup4
    • Scrapy
    • Selenium

BeatifulSoup4

  • An HTML parser
  • Source code obtained by a request is passed to BS4 for easier parsing
  • Has troubles with pages that load the content dynamically with JavaScript
    • After the request is made, an empty page is returned
    • The content dynamically loads later, which is not represented in what we retrieved
  • Full web crawling and parsing example using BS4, requests and much more can be found here

Scrapy

  • Scrapy is a whole scraping framework instead of a simple Python library
  • After installation, we obtain a command line tool called scrapy
  • We first need to generate a Scrapy project (scrapy startproject project_name)
  • Then we generate a spider/scraper (scrapy genspider name url_to_scrape)
  • Each spider has a parse method which accepts the request's response
  • In the parse method, we work essentially the same way as we would work with BS4
  • Managing large parsing projects definitely gets easier than with pure Python, but is a little cumbersome for small applications/one-time parsers
  • Still doesn't solve the problem with dynamically-generated content (by default)

Scraping dynamically-generated content

  • Reverse engineering JavaScript
    • Basically, looking at the source code of the website and trying to figure out how does the page load the content (e.g. what sources does it use)
    • If this works out, then one can easily use the same source to load the data
    • Can be very time consuming, but then working with the source data is much easier
  • Pre-rendering JavaScript
    • Installing a headless browser that acts like a middle man between you and the server to pre-render JavaScript before response is received
    • The browser receives the reponse, waits for the dynamical content to load and then sends the loaded content to you
    • Higher effectivity than trying to reverse engineer the JavaScript, but can heavily slow your code

Selenium

  • A framework for testing web applications
  • Can also be used for web scraping as it simulates user actions inside a browser, and therefore dynamically generated content is retrieved as well
  • To use Selenium, one has to have a browser and its driver of the same version installed (e.g. Chrome and Chromedriver)
  • Once a driver is loaded, the browser opens and one can navigate through it using several commands
    • .get will open a web page
    • .click will click on currently chosen element
    • .send_keys will write into currently chosen element
    • .find_element_by_id and .find_elements_by_class_name will search for an element by either its ID or class, respectively
